# Urban Routes – Sprint 7
### Manual Web Testing Project (TripleTen QA Engineering Program)

This project is part of TripleTen’s QA Engineering program, Sprint 7.  
It focuses on performing **manual web testing** of the Urban Routes application, including test case design, functional validation, UI checks, bug reporting, and documentation.

This repository contains the clean and finalized version of the Sprint 7 work, following the project instructions and QA best practices.

---

## Project Overview

Urban Routes is a web-based taxi ordering service.  
In Sprint 7, the focus was on:

- Reviewing application requirements
- Creating a complete set of manual test cases
- Testing core UI and functional components
- Verifying form behavior, input validation, and state changes
- Testing multiple browser types (Chrome and Opera)
- Reporting defects discovered during testing
- Documenting expected vs. actual behavior
- Preparing a clear submission package

This sprint introduces the fundamentals of **manual QA**, including structured testing, requirement analysis, and defect communication.

---

## Key Deliverables

### Test Cases
A full set of test cases designed in Google Sheets, including:

- Positive scenarios  
- Negative scenarios  
- Edge cases  
- UI/UX validation  
- Functional validation  
- Field-level test coverage  

Each test case includes:

- Test ID  
- Title  
- Preconditions  
- Test steps  
- Expected results  
- Actual results  
- Pass/Fail status  
- Links to bugs (if applicable)
